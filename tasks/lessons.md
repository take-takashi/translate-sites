# Lessons

- README を正本とし、概要変更時は `openspec/project.md` へ同期する。
- 最初の対象サイトを決めたら、OpenSpec の proposal・design・spec・tasks すべてに受け入れ対象として明示する。
- ローカル専用設定を ignore したい場合は、`.gitignore` 追加だけでなく追跡済みかどうかを先に確認する。
- ユーザーが求めているのが CLI か運用ワークフローかを先に切り分け、実装形態を早い段階で固定する。
- 仕様が複数ファイルに分かれたら、`openspec/specs/SPECS.md` のような目次を用意して参照順と役割を明示する。
