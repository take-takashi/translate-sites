## 背景

このプロジェクトで必要なのは単発の翻訳CLIではなく、翻訳対象サイトを設定ファイルで管理し、取得済み原文と翻訳済み成果物を継続的に整理できる翻訳管理ワークフローです。`mise` のサンプルを1件作ったことで、次に必要なのは raw / ja / state を一貫して扱う共通ルールの仕様化だと明確になりました。

## 変更内容

- サイトごとの設定ファイルで翻訳対象URLと保存先を定義する要件を追加する
- 設定ファイルに基づいてページを raw ディレクトリへ保存する要件を追加する
- raw から翻訳済み Markdown を ja ディレクトリへ保存する要件を追加する
- ページ単位の取得状態・翻訳状態・追跡情報を state ファイルへ保存する要件を追加する

## Capability

### 新規 Capability
- `site-config-registry`: サイト設定ファイルで対象URL、保存先、許可ドメインを管理する
- `raw-page-storage`: 設定ファイルから特定したページを raw ディレクトリへ保存する
- `translated-page-storage`: raw を元に翻訳済み Markdown を ja ディレクトリへ保存する
- `translation-state-tracking`: 取得・翻訳・追跡情報を site 単位の state ファイルに記録する

### 変更対象 Capability

## 影響範囲

- `sites/`, `data/raw/`, `data/ja/`, `data/state/` を中心にした保存構造の整理が必要になる
- 単発CLI前提ではなく、設定ファイル駆動のワークフローへ設計を切り替える必要がある
- 将来的な更新追従や再翻訳の基盤として state 管理の導入が必要になる
