## Why

このプロジェクトには、英語のWebページを取得して日本語のMarkdownとして保存するための実装がまだ存在しません。最初の価値を早く検証するため、単一URLを対象にした最小構成のCLIを定義し、本文抽出と翻訳の品質要件を先に固める必要があります。

最初の検証対象として `mise-en-place` のドキュメントサイトを扱うことで、技術文書に多い見出し、リンク、コードブロックを含むページ構造に対して、MVPの品質を現実的に確認できます。

## What Changes

- 単一のURLを入力として受け取り、ページ取得からMarkdown保存までを一連で処理するCLIの要件を追加する
- 本文抽出、Markdown変換、日本語翻訳、ファイル保存の各段階で満たすべき振る舞いを仕様化する
- 出力ファイルに元URLなどの追跡可能なメタ情報を保持する要件を追加する
- `mise-en-place` の代表的なドキュメントページを使って、初期実装の受け入れ確認方針を明確にする

### New Capabilities
- `single-page-translation-cli`: 単一URLのWebページを取得し、日本語Markdownとして保存するCLIの振る舞いを定義する

## Impact

- 新規に CLI エントリポイントとページ処理パイプラインの実装が必要になる
- HTML取得、本文抽出、Markdown変換、翻訳処理のための依存ライブラリ追加を伴う可能性がある
- 出力ディレクトリと保存形式の設計、ならびに検証用テストの追加が必要になる
- 最初の検証では `mise-en-place` のトップページと導入ページをサンプルとして扱う
