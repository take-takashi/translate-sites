# Project Context

## Purpose

このプロジェクトは、主に OSS、ライブラリ、フレームワークなどの英語の技術系Webサイトを取得し、原文と日本語訳をサイト単位で管理できる翻訳管理アプリを整えることを目的とします。

人が読みやすく、あとで編集しやすいMarkdownを生成することを重視します。詳細な背景や方針は [README.md](../README.md) を正本として参照します。

## Current Focus

- 最初のMVPは設定ファイルでサイトを定義し、raw / ja / state を分離して管理する
- 処理は `config -> download -> translate -> store -> track` の流れを基本とする
- まずは対象サイトを迷わず管理できることを優先する
- 取得済み原文と翻訳済み成果物を再利用できる保存構造を優先する

## Non-Goals

- サイト全体の大規模クロール
- 認証が必要なページへの対応
- 完全自動の高品質な意訳
- 画像や動画など全アセットの保存
- 独自CMSの構築
- 汎用CLI製品としての磨き込み

## Technical Direction

- 設定ファイル、raw保存、翻訳済み保存、状態管理を分離する
- 実装より先に、どのファイルをどこへ置くかの管理モデルを固める
- 取得処理と翻訳処理は将来差し替え可能な構成を優先する
- raw から ja を再生成できるよう、翻訳前の入力を保持する

## Quality Bar

- 本文以外のノイズをできるだけ除去する
- 見出し、リンク、リスト、コードブロックなどのMarkdown構造を壊さない
- raw / ja / state の対応関係を追跡可能にする
- 小さく検証可能な単位で実装する
- サイト単位の構成が分かりやすいことを保つ
