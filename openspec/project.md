# Project Context

## Purpose

このプロジェクトは、主に OSS、ライブラリ、フレームワークなどの英語の技術系Webサイトを取得し、日本語のMarkdownとしてサイト単位に整理して保存し、GitHub Pages で閲覧できるようにすることを目的とします。

人が読みやすく、あとで編集しやすいMarkdownを生成することを重視します。詳細な背景や方針は [README.md](../README.md) を正本として参照します。

## Current Focus

- 最初のMVPは単一URLを対象にする
- 処理は `fetch -> extract -> convert -> translate -> save` のパイプラインを基本とする
- まずは1ページを安定してきれいに保存できることを優先する
- 保存先は将来的なサイト別ディレクトリ構成と GitHub Pages 公開を見据えて設計する

## Non-Goals

- サイト全体の大規模クロール
- 認証が必要なページへの対応
- 完全自動の高品質な意訳
- 画像や動画など全アセットの保存
- 独自CMSの構築

## Technical Direction

- Node.js ベースのCLIとして実装する
- HTML取得、本文抽出、Markdown変換、翻訳、保存を段階ごとに分離する
- 各段階は将来差し替え可能な構成を優先する
- 翻訳前に本文抽出とMarkdown変換を行い、構造維持をしやすくする
- 将来的にサイトごとの出力整理と更新追従に拡張しやすい保存形式を選ぶ

## Quality Bar

- 本文以外のノイズをできるだけ除去する
- 見出し、リンク、リスト、コードブロックなどのMarkdown構造を壊さない
- 出力に元URLを残し、追跡可能にする
- 小さく検証可能な単位で実装する
- 公開時にサイト単位の構成が分かりやすいことを保つ
