# translate-sites

英語で提供されているテック系Webサイトの内容を取得し、日本語化したうえで、取得済みコンテンツと翻訳済みコンテンツをサイト単位で管理するためのプロジェクトです。

## プロジェクト方針

このリポジトリは、主に OSS、ライブラリ、フレームワーク、技術ドキュメントなどの英語コンテンツを「読みやすい日本語のMarkdown」として再利用しやすくすることを目的とします。

単にHTMLを翻訳するのではなく、対象URLを設定ファイルで管理し、取得済み原文と翻訳済み成果物を分けて保持し、あとで更新追跡や再翻訳をしやすい形に整えることを重視します。

## 目指すもの

- 翻訳対象サイトと開始URLを設定ファイルで管理できる
- 設定ファイルに基づいて対象ページを raw フォルダへ取得できる
- 取得済みファイルを元に日本語翻訳を行い、ja フォルダへ保存できる
- 元URLや取得時刻、翻訳状態などを追跡できる
- サイトごとに raw / ja / state を分けて管理できる

## 最初のMVP

最初は、設定ファイルを1つ定義し、その設定に従って対象ページを取得して保存し、保存済みファイルを翻訳して別フォルダへ出力する最小構成を作る方針とします。

1. サイト設定ファイルを1つ用意する
2. 設定ファイルから対象URLを取得する
3. 対象ページを raw ディレクトリへ保存する
4. raw ディレクトリのファイルを翻訳する
5. 翻訳結果を ja ディレクトリへ保存する
6. 状態ファイルで取得・翻訳状況を追跡する

この段階では、まず「対象サイトを迷わず管理できること」と「取得済み原文と翻訳済み成果物を分離して保持できること」を優先します。

## 非目標

初期段階では、以下は必須にしません。

- サイト全体の大規模クロール
- 完全自動の高品質な意訳
- 認証が必要なページへの対応
- 画像や動画など全アセットの保存
- 独自CMSや汎用CLI製品としての完成

## 想定する処理フロー

```text
設定ファイル
  ↓
対象URL決定
  ↓
raw として保存
  ↓
翻訳対象抽出
  ↓
日本語翻訳
  ↓
ja として保存
  ↓
state 更新
```

## 品質上の重視点

- 本文以外のノイズをできるだけ除去すること
- 見出し、リンク、リスト、コードブロックなどの構造を壊さないこと
- 後から人間が手で修正しやすい出力にすること
- サイトごとの raw / ja / state の対応関係が分かりやすいこと
- 処理を小さく分割し、取得・翻訳・状態管理を独立に扱えること

## 今後の拡張候補

- サイト内リンクをたどるクロール
- 翻訳キャッシュ
- 用語集による訳語の統一
- 更新検知による差分更新や再取得の最適化
- 日々更新されるサイトのバージョン追従
- GitHub Pages 向けの公開ディレクトリ生成

## 想定ディレクトリ

```text
sites/
  mise.yml

data/
  raw/
    mise/...
  ja/
    mise/...
  state/
    mise.json
```

設定ファイル形式は [`sites/README.md`](sites/README.md)、保存構造は [`data/README.md`](data/README.md) を正本とします。
