# translate-sites

英語で提供されているテック系Webサイトの内容を取得し、日本語化したうえでサイト単位に整理して保存し、GitHub Pages で閲覧できる形にするためのプロジェクトです。

## プロジェクト方針

このリポジトリは、主に OSS、ライブラリ、フレームワーク、技術ドキュメントなどの英語コンテンツを「読みやすい日本語のMarkdown」として再利用しやすくすることを目的とします。

単にHTMLを翻訳するのではなく、本文を抽出し、文書構造を保ち、サイトごとにフォルダ分けして、あとで編集しやすく公開しやすい形に整えることを重視します。

## 目指すもの

- 対象のサイトURLまたはページURLから内容を取得できる
- ナビゲーションやフッターではなく、本文中心のコンテンツを抽出できる
- 文書構造を保ったまま Markdown に変換できる
- Markdown の構造を壊さずに英語から日本語へ翻訳できる
- 元URLなどのメタ情報とあわせてサイトごとのフォルダに `.md` ファイルとして保存できる
- 生成した成果物を GitHub Pages で閲覧できる形に配置できる

## 最初のMVP

最初は、単一の英語ページを対象にした最小構成のCLIを作る方針とします。将来的なサイト単位の展開を見据えつつ、まずは 1 ページを安定して日本語化できることを優先します。

1. URLを1つ受け取る
2. ページを取得する
3. 本文を抽出する
4. Markdown に変換する
5. 日本語へ翻訳する
6. サイト名ベースのディレクトリ配下に保存する

この段階では、まず「安定して1ページをきれいに落とせること」を優先します。

## 非目標

初期段階では、以下は必須にしません。

- サイト全体の大規模クロール
- 完全自動の高品質な意訳
- 認証が必要なページへの対応
- 画像や動画など全アセットの保存
- 独自CMSの構築

## 想定する処理フロー

```text
URL
  ↓
HTML取得
  ↓
本文抽出
  ↓
Markdown変換
  ↓
日本語翻訳
  ↓
サイト別に保存
  ↓
GitHub Pages で公開
```

## 品質上の重視点

- 本文以外のノイズをできるだけ除去すること
- 見出し、リンク、リスト、コードブロックなどの構造を壊さないこと
- 後から人間が手で修正しやすい出力にすること
- サイトごとのディレクトリ構造が把握しやすいこと
- 処理を小さく分割し、差し替え可能な構成にすること

## 今後の拡張候補

- サイト内リンクをたどるクロール
- 翻訳キャッシュ
- 用語集による訳語の統一
- frontmatter の付与
- 更新検知による差分更新や再取得の最適化
- 日々更新されるサイトのバージョン追従
